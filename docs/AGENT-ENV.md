# Аудит окружения pi под Qwen 3.6-35B-A3B (oQ4) — до уровня «Opus-замены»

> Исторический срез. Текущий профиль `ThinkingCap-Qwen3.6-27B-oQ4e-DWQ-MTP-Vision`, окно 262,144 и новый workflow описаны в [HARNESS-REDESIGN-2026-07-18.md](HARNESS-REDESIGN-2026-07-18.md).

**Дата:** 2026-07-02 · Модель: Qwen3.6-35B-A3B-Claude-4.7-Opus-Reasoning-Distilled, MLX oQ4, 128k ctx, MTP.
Замеры сделаны на живых сессиях pi 0.80.3 через RPC.

## 1. Главный факт: стартовый промпт ≈ 40 000 токенов

В реальной сессии первый же ответ ассистента имеет `input: 16 007 + cacheRead: 24 576 ≈ 40.5k` токенов.
**Треть контекста (31% от 128k) занята до первого слова пользователя.** Для q4-кванта это двойной
удар: меньше места для работы и больше «шума инструкций», в котором 35B теряет фокус и уходит
в лупы. Целевой бюджет старта: **≤ 15k токенов**.

Откуда набегает:
| Источник | Оценка | Комментарий |
|---|---|---|
| 20 npm-расширений (инструкции + тулы) | ~15–20k | каждое добавляет системные инструкции и/или тулы |
| 13 каталогов skills (`~/.claude/**`) | ~8–12k | фронтматтеры сотен SKILL.md из fullstack/devops-паков |
| 15 sub-agent определений (`agents/*.md`) | ~3–5k | pi-subagents публикует их описания |
| AGENTS.md + системный промпт pi | ~2k | ок |
| MCP-адаптер (context7, seq-thinking) | ~1k | лениво, ок |

## 2. Пакеты: вердикты (20 установлено)

**Ядро — оставить глобально (≈9):**
`pi-claude-style-tools`, `@gotgenes/pi-permission-system`, `@plannotator/pi-extension`,
`@sting8k/pi-vcc` (алгоритмическая компакция без LLM — правильно для локалки),
`@narumitw/pi-retry`, `@narumitw/pi-statusline`, `@juicesharp/rpiv-todo`,
`@juicesharp/rpiv-ask-user-question`, `pi-mcp-adapter`.

**Перевести в per-workspace (включать в `<проект>/.pi/settings.json` только там, где нужны):**
- `pi-web-access` — нужен в исследовательских задачах, не в каждом чате;
- `@tintinweb/pi-subagents` — сабагенты на одной локальной GPU выполняются
  последовательно и плодят контекст; 15 описаний агентов висят в промпте всегда.
  Включать точечно + сократить `~/.pi/agent/agents/` до 3–5 реально используемых;
- `pi-lens` — очень полезен в кодовых проектах, но тяжёлый (LSP+линтеры+тесты).
  Держать в проектах с кодом, отключать `--no-lens` в «болталках»;
- `pi-rewind`, `@mrclrchtr/supi-cache`, `@spences10/pi-observability`,
  `@spences10/pi-team-mode` (в RPC-зонде виден постоянный поток team-виджетов),
  `@amaster.ai/pi-task-scheduler`, `@trevonistrevon/pi-loop` — фоновые/нишевые,
  на GPU-арбитре должны жить только по требованию.

**Дубль — убрать один:** `@aliou/pi-guardrails` и `pi-permission-system` оба гейтят
опасные команды. Permission-system функциональнее (пресеты режимов pi-app работают
через него) → guardrails можно снять: `pi remove npm:@aliou/pi-guardrails`.

Готовые команды:
```bash
pi remove npm:@aliou/pi-guardrails
pi remove npm:@spences10/pi-team-mode        # вернуть per-project при парной работе
pi remove npm:@trevonistrevon/pi-loop        # вернуть при необходимости
# per-workspace включение (пример): <proj>/.pi/settings.json
# { "packages": ["npm:pi-web-access", "npm:@tintinweb/pi-subagents"] }
```

## 3. Skills: 13 каталогов → 4

`~/.claude/plugins/marketplaces/fullstack-dev-skills/**` и `devops-skills` — это сотни
скиллов уровня «react-expert/golang-pro/...». Для 35B это токены и рассеивание внимания:
модель видит 60+ названий и «хочет» их применять. Оставить глобально:
```json
"skills": [
  "~/.claude/skills",
  "~/.claude/plugins/marketplaces/claude-plugins-official/plugins/skill-creator/skills"
]
```
Остальные паки подключать per-project по факту (`.pi/settings.json` → `skills`).

## 4. AGENTS.md — добавлены правила для q4 и 128k (применено, бэкап рядом)

Добавлена секция «Long context & q4 discipline» (см. diff в `AGENTS.md.bak-pi-app`):
1. один инструмент за шаг при неуверенности; после 2 неудачных повторов — стоп и вопрос;
2. после авто/ручной компакции: перечитать todo-список и сформулировать текущее состояние
   одним абзацем, прежде чем продолжать (якорь против пост-компакт дрейфа);
3. при >60% заполнения контекста — приоритет `compact` перед новым большим чтением;
4. большие файлы читать выборочно (`rg` → диапазоны строк), не целиком.

## 5. RAG / память — что уже есть и что добавить

Уже стоит и покрывает 80% потребности:
- **pi-hermes-memory** — персистентная память проектов + поиск по сессиям (это и есть
  локальный RAG по вашей истории; настройте `/memory-index-sessions` разово);
- **context7 MCP** — RAG по актуальным докам библиотек (вместо галлюцинаций API);
- **sequential-thinking MCP** — внешняя «доска рассуждений», разгружает контекст.

Рекомендация к добавлению (по одному, с замером):
- **`pi-readseek`** — hash-anchored read/edit + структурные карты кода: резко снижает
  токены на чтение больших файлов (главная статья расхода на 128k);
- альтернатива: `pi-shazam` (7 структурных инструментов анализа кодбейзы).
Не ставить оба сразу: «fewest tools wins» для 35B важнее функциональности.

## 6. Компакция и лупы — настройки

- Порог авто-компакции держать ~70–75% (не 90%): q4 деградирует на хвосте контекста
  раньше, чем кончится окно.
- `pi-vcc` алгоритмический компактор — правильный выбор (не тратит токены модели);
  после компакции работает правило №2 из AGENTS.md.
- `pi-retry` уже стоит: stall-timeout 90s покрывает зависание стрима.
- Признак лупа для 35B: повтор одинакового tool-call. Permission-system в режиме
  Ask/Accept-edits прерывает такие циклы диалогом — ещё одна причина не жить в bypass.

## 7. Итоговая целевая конфигурация

| Параметр | Сейчас | Цель |
|---|---|---|
| Стартовый промпт | ~40k | ≤ 15k |
| Глобальных расширений | 20 | 9–11 |
| Skills-каталогов глобально | 13 | 2–4 |
| Sub-agent определений | 15 | 3–5 |
| Авто-компакция | по умолчанию | порог ~70–75% |

После сокращения промпта у модели остаётся ~110k рабочего контекста, меньше
конкурирующих инструкций и — по опыту q4-квантов — заметно ниже частота лупов.
Изменения в `settings.json` намеренно **не применены автоматически** (набор пакетов —
ваш выбор): команды выше выполняются за минуту, всё обратимо через `pi install`.
