# pi-app — нативное macOS-приложение на базе pi.dev (замена Claude Desktop)

**Версия плана:** 1.0 · 2026-07-02
**Целевая установка pi:** `~/.pi/agent`, pi v0.80.3, локальная модель Qwen3.6-35B (oMLX, GPU-арбитр на `:8099`), 20 npm-расширений, внешние каталоги skills из `~/.claude`.

---

## 1. Цель и позиционирование

Полноценная 1:1 замена Claude Desktop (Claude for Mac) поверх pi.dev:

- **pi — это движок.** Приложение не реализует агента заново: оно запускает установленный `pi --mode rpc` и говорит с ним по JSONL-протоколу. Все расширения, skills, темы, модели, сессии и auth пользователя подхватываются автоматически (как у pi-lot: «detects existing pi user sessions, settings, and authentication»).
- **Дизайн и продуктовая модель — от pi-lot.dev (приоритет 1):** workspaces с изоляцией проектов, реалтайм-стриминг tool calls, дерево сессий с форками, аналитика (стоимость/токены/heatmap), diff/review как первоклассная фича.
- **Заимствуем у tau:** очередь сообщений во время работы агента, полнотекстовый поиск по истории, голосовой ввод, live-индикаторы сессий, инлайн-diff в tool calls, компакция из UI.
- **Заимствуем у pi-gui:** мульти-workspace-модель, timeline каждого действия агента с полным вводом/выводом, абстракция `SessionDriver` (фронтенд не знает о транспорте).
- **Паритет с Claude Desktop:** глобальный быстрый ввод (⌥Space), пункт в menu bar, диктовка, нативные уведомления, connectors (у нас — MCP + расширения pi), автообновление, проекты.

Не-цели v1: Windows/Linux, собственный биллинг/аккаунты, мобильные клиенты, встроенный текстовый редактор кода (для этого — интеграция с VS Code и др.).

## 3. Архитектура

### 3.1 Стек

**Tauri 2 (Rust-ядро + системный WKWebView) + TypeScript/React 19 + Vite.**

Почему не Electron (путь pi-gui): бинарник ~15–25 МБ против 150+ МБ, RAM idle ~150 МБ против 400+, нативный WKWebView с ProMotion-скроллом. Почему не чистый SwiftUI: чат с потоковым markdown, подсветкой кода и diff-вьювером на порядок быстрее и качественнее собирается на веб-стеке; нативность добираем Rust/Swift-плагинами Tauri (menu bar, глобальные хоткеи, диктовка, вибранси).

Ключевые библиотеки фронтенда:
- **Виртуализация:** `virtua` или `react-virtuoso` (чат, списки сессий, diff).
- **Markdown-стриминг:** инкрементальный парсер (`streaming-markdown` / собственный на `marked` с дельта-аппендом), без ре-парсинга всего сообщения на каждый чанк.
- **Подсветка:** `shiki` в Web Worker, кэш по хэшу блока.
- **Diff:** `@git-diff-view/react` (side-by-side + inline, word-level).
- **Анимации:** `motion` (Framer Motion) + CSS-переходы; графики аналитики — `visx`/`echarts` (lazy chunk).
- **JSON-редактор конфигов:** CodeMirror 6 + JSON Schema валидация.

Rust-ядро (crate `pi-host`):
- Супервизор процессов pi (spawn/kill/health, лимиты, авто-перезапуск).
- Строгий JSONL-фреймер (LF-only, срез trailing `\r` — требование RPC-спеки, обычный readline не подходит), backpressure, батчинг событий в UI-мост (сгруппированный emit ≤ раз в 16 мс).
- Индексатор сессий: SQLite (FTS5) с инкрементальным вотчем `~/.pi/agent/sessions` (метаданные, имена, cwd, стоимость, полнотекст).
- Git-слой для code review: libgit2 (`git2-rs`) — снапшоты, диффы, статусы.
- FS-вотчеры конфигов (`notify`), чтение/атомарная запись settings/models/mcp с бэкапом.

### 3.2 Процессная модель

```
pi-app (Tauri)
├─ Rust core: supervisor · JSONL codec · SQLite index · git · fs-watch
│    ├─ pi --mode rpc  (workspace A, активная сессия)
│    ├─ pi --mode rpc  (workspace B)         ← lazy: спавн при первом промпте
│    └─ pi install/remove/update/config …    ← короткоживущие CLI-вызовы
└─ WebView (React): Chat · Sessions · Review · Config · Analytics
```

- **Один RPC-процесс на активную сессию workspace**, `cwd` = папка workspace. Спавн ленивый, при первом сообщении.
- **Критично для этой машины (48 GB, GPU-арбитр, single-model swap):** конфигурируемый лимит параллельных pi-процессов (по умолчанию 1–2), idle-kill через N минут без активности (состояние восстанавливается через `--session <id>`), никаких фоновых поллинг-циклов, удерживающих модель. Просмотр истории — только чтение JSONL, процесс не нужен.
- `SessionDriver`-абстракция во фронтенде (как у pi-gui): UI знает только команды/события, транспорт (локальный RPC, в будущем SSH/remote) — деталь реализации.

### 3.3 RPC-интеграция (контракт с pi ≥ 0.80)

Используемые команды: `prompt` (с изображениями и `streamingBehavior`), `steer`, `follow_up`, `abort`, `new_session`, `switch_session`, `fork`, `clone`, `get_fork_messages`, `set_session_name`, `get_state`, `get_messages`, `get_entries` (курсор — для догрузки истории), `get_tree`, `set_model`, `get_available_models`, `set_thinking_level`, `compact`, `set_auto_compaction`, `set_auto_retry`, `get_session_stats`, `export_html`, `get_commands`, `bash`.

Обрабатываемые события: `agent_start/end`, `turn_start/end`, `message_start/update/end` (дельты text/thinking/toolcall), `tool_execution_start/update/end`, `queue_update`, `compaction_start/end`, `auto_retry_start/end`, `extension_error`, `extension_ui_request`.

**Мост Extension UI (обязателен — у пользователя 20 расширений):** `extension_ui_request` рендерится нативно: `select`/`confirm`/`input`/`editor` — модальные шиты с таймаут-индикатором; `notify` — тост/уведомление; `setStatus` — статус-бар сессии; `setWidget` — закреплённая карточка над полем ввода; `set_editor_text` — префилл инпута. Это даёт «бесплатную» поддержку установленных permission-prompts (`pi-permission-system`, guardrails), plan-mode (plannotator), todo, ask-user-question и т.д.

Слэш-команды и prompt templates из `get_commands` — в палитру `/` автодополнения поля ввода.

### 3.4 Данные

- **Источник правды — файлы pi.** Приложение не дублирует состояние: сессии `~/.pi/agent/sessions/<slug-cwd>/<ts>_<uuid>.jsonl` (v3, tree по `parentId`), `settings.json`, `models.json`, `mcp.json`, `auth.json` (никогда не показываем содержимое, только статус «настроено»).
- **Свой слой — только кэш/индекс:** SQLite в `~/Library/Application Support/pi-app/` (FTS-индекс сессий, агрегаты аналитики, review-чекпоинты, UI-состояние). Удаление кэша безвредно — пересоберётся.
- Настройки самого приложения — `config.json` там же (хоткеи, редактор, лимиты процессов, тема UI).

## 4. Функциональные требования — ядро

### 4.1 Чат
- Стриминг текста, «толстые» блоки thinking (сворачиваемые, приглушённые), tool calls как живые карточки: аргументы → стриминг вывода → результат/ошибка; для `edit`/`write` — инлайн-дифф прямо в карточке (как у tau).
- **Steering/queue:** Enter во время работы агента = steer, ⌥Enter = follow-up; видимая очередь с возможностью убрать сообщение (`queue_update`); режимы very `one-at-a-time`/`all`.
- Abort (Esc), авто-retry с индикатором и кнопкой отмены (`auto_retry_*`).
- Вложения: изображения (paste/drag/кнопка), @-упоминания файлов workspace с fuzzy-поиском, drag файла из Finder → путь.
- Палитры: `/` — команды расширений + шаблоны + skills; `⌘K` — глобальная палитра действий.
- Индикатор контекста: заполненность окна, токены/стоимость сессии (`get_session_stats`), кнопка «Compact» + авто-компакция с событиями прогресса.
- Копирование сообщения/блока кода в один клик, «новые сообщения ↓» при отрыве от низа, диктовка.

### 4.2 Workspaces и сессии
- Workspace = папка проекта; сайдбар со списком, цветами/иконками, мгновенное переключение (состояние чатов сохраняется в памяти UI).
- Список сессий по workspace: сортировка по изменению, live-точка активных, переименование инлайн, теги/избранное (в нашем SQLite), read-only просмотр без спавна pi.
- **Дерево сессий:** визуальный граф веток (fork из любого user-сообщения через `get_fork_messages`/`fork`, `clone` ветки), навигация как в pi-lot.
- Глобальный FTS-поиск по всем сессиям всех проектов с подсветкой и прыжком к сообщению.
- Экспорт: HTML (`export_html`), markdown, share-ссылка gist (если настроен gh).
- Онбординг: автообнаружение `~/.pi/agent`; если pi не установлен — экран установки (brew/curl) с проверкой версии (минимум 0.80).

### 4.3 Модели и провайдеры
- Селектор моделей (searchable, клавиатурная навигация) из `get_available_models` + `models.json`; быстрый цикл ⌘P; thinking level слайдером off→xhigh.
- Отображение возможностей модели (text-only/vision, context window) — валидация вложений до отправки.

### 4.4 Аналитика (pi-lot-style)
- Дашборд per-workspace и глобально: стоимость, input/output/cache-токены, число сессий/turn'ов, разбивка по моделям, **heatmap активности по дням**.
- Источник — агрегация JSONL + `get_session_stats`; всё локально.

## 6. Config Center — управление pi из приложения

Раздел Settings с живой валидацией и атомарной записью (tmp + rename, автобэкап предыдущей версии, как pi делает `settings.json.bak-*`). Каждый экран имеет **Form-режим и Raw JSON-режим** (CodeMirror + JSON Schema, diff перед сохранением).

1. **General (`settings.json`):** defaultProvider/defaultModel/defaultThinkingLevel, theme, прочие ключи — форма + raw.
2. **Packages / Extensions:** список из `settings.packages` с метаданными npm (версия, описание, ссылка); установка/удаление через `pi install|remove <source>` (стрим вывода в UI); `pi update --all` с ченджлогом; enable/disable ресурсов пакета — GUI-эквивалент `pi config`; каталог пакетов pi.dev/packages со встроенным поиском и установкой в 1 клик; отдельно — локальные расширения `~/.pi/agent/extensions` и project-local.
3. **Skills:** объединённый список из всех `settings.skills`-директорий (включая `~/.claude/...`), просмотр SKILL.md, добавление/удаление каталогов, создание скелета скилла, per-workspace skills.
4. **MCP (`mcp.json`):** карточки серверов (command/args/lifecycle/directTools), добавить/удалить/выключить, тест подключения, стрим логов.
5. **Models (`models.json`):** редактор провайдеров и моделей (baseUrl, api, contextWindow, sampling, extraBody) — критично для локального oMLX-сетапа; пинг baseUrl.
6. **Auth:** статусы ключей по провайдерам (env/`auth.json`), запуск логин-флоу pi; секреты не отображаются.
7. **Themes:** импорт тем pi, синхронизация темы UI приложения с темой pi, свои пресеты.
8. **App:** хоткеи, лимит одновременных pi-процессов, idle-kill таймаут, выбор внешнего редактора, автообновление.

Изменения конфигов вне приложения подхватываются вотчером; при конфликте — предложение перечитать/смержить. Изменения, требующие перезапуска RPC-процесса, помечаются и применяются по кнопке «Restart agent».

## 7. Code Review — встроенный, с выходом в VS Code

Главное отличие от Claude Desktop.

### 7.1 Трекинг изменений агента
- **Чекпоинты:** перед каждым turn'ом Rust-ядро фиксирует снапшот workspace (git stash-free: временный commit в теневом ref `refs/pi-app/checkpoints/*` либо shadow-index при отсутствии git — предлагаем `git init`).
- Live-лента изменённых файлов сессии (из `tool_execution_end` для edit/write/bash + git status), бейдж «N files changed, +A −B» в шапке чата.

### 7.2 Review-режим
- Полноэкранный вид: слева — дерево изменённых файлов с статистикой, справа — diff (side-by-side / inline, word-level, подсветка shiki, сворачивание контекста, виртуализация для больших диффов).
- Диапазоны сравнения: «с начала сессии», «с последнего чекпоинта/turn'а», «между любыми чекпоинтами», «uncommitted vs HEAD».
- **Комментарии к строкам/ханкам** → одна кнопка «Send to agent»: комментарии собираются в структурированный промпт и уходят как steer/follow-up (замыкает цикл ревью, идея plannotator).
- Действия по ханку/файлу: **Revert** (к чекпоинту), Stage, Copy patch; финал: сформировать commit (message предлагает агент) — только по явной команде пользователя.
- Интеграция с установленными ревью-расширениями (plannotator, pi-lens findings) через Extension UI-мост.

### 7.3 Открытие во внешнем редакторе
- Конфигурируемый редактор: VS Code (`code --goto file:line`, `code --diff a b`), Cursor, Zed, JetBrains (`idea --line`), Sublime; авто-детект установленных.
- Из любого места: файл в карточке tool call, строка диффа, файл в дереве — «Open in VS Code» (⌘↩) с точной позицией; «Open workspace in editor».
- Fallback: `open -R` в Finder, Quick Look-предпросмотр.

## 8. Требования к дизайну

- **HIG-first:** нативный тайтлбар с traffic lights поверх контента (`titleBarStyle: overlay`), сайдбар с вибранси (`NSVisualEffectView` через Tauri-плагин), SF Pro / SF Mono, системные размеры и отступы, ⌘, для настроек, полный набор стандартных шорткатов.
- **Макет (по pi-lot):** трёхпанельный — сайдбар (workspaces + сессии, сворачиваемый ⌘\), центр (чат/review/аналитика), правый инспектор по требованию (файлы workspace, статистика сессии, очередь). Табы сессий внутри workspace.
- **Темы:** light/dark вслед за системой + акценты; импорт цветов из тем pi, чтобы TUI и GUI совпадали.
- **Типографика чата:** текст 13–14 pt, код SF Mono/JetBrains Mono 12 pt, ширина колонки чата ограничена (~76ch), thinking — приглушённый цвет + меньший кегль.
- **Анимации — быстрые и физичные, никогда не блокирующие ввод:**
  - появление сообщений/карточек: spring 200–250 мс, opacity+translateY(4px);
  - стриминг текста: без по-символьных твинов — рендер чанков как есть (скорость = скорость модели), мягкий каретка-индикатор;
  - tool call карточки: расширение по высоте spring ~220 мс; статус-иконка crossfade 120 мс;
  - переключение workspace/сессий: crossfade 150 мс, без layout-прыжков (скелетоны под высоту);
  - review-панель: slide-in 240 мс; сайдбар: width spring 200 мс;
  - уважать `prefers-reduced-motion`; цель 120 Гц на ProMotion.
- **Состояния:** продуманные empty states (нет workspace / нет сессий / агент не установлен), skeleton-загрузка, inline-ошибки с retry, offline-режим (история доступна всегда).
- **Доступность:** полная клавиатурная навигация, VoiceOver-лейблы, контраст WCAG AA, масштабирование текста ⌘+/−.

## 10. Системные требования

- **ОС:** macOS 13 Ventura+ (вибранси/нотификации/современный WKWebView); universal binary arm64 + x86_64, приоритет Apple Silicon.
- **Зависимости:** установленный pi ≥ 0.80 (`pi` в PATH или `~/.pi`), Node ≥ 20 (рантайм pi; проверяется онбордингом), git ≥ 2.39 для review-функций (иначе review деградирует до «diff за сессию» без чекпоинтов).
- **Ресурсы:** ~60 МБ диска (app) + кэш-индекс (~1–50 МБ); RAM: 180 МБ app + память pi-процессов; для локальной модели пользователя — учёт GPU-арбитра (лимит процессов, раздел 3.2).
- **Дистрибуция:** подписанный и нотарифицированный .dmg (Developer ID + hardened runtime), Tauri updater, Homebrew cask. Entitlements: без App Sandbox в v1 (нужен полный FS/exec-доступ), микрофон (диктовка), уведомления, Accessibility не требуется.
- **Приватность:** сеть — только к настроенным провайдерам pi и npm (установка пакетов); телеметрии нет; все данные локальны.

## 12. Риски и меры

| Риск | Мера |
|---|---|
| Изменения RPC-протокола pi (0.x, движется быстро) | Тонкий адаптер-слой + контрактные тесты против установленной версии; проверка версии при старте; `SessionDriver` изолирует UI |

## 13. Итерация 2026-07-03 — live-сессии, git-центр, контекст-виджет (выполнено)

2. **Alignment под инпутом:** statusline стал однострочным (`nowrap` + ellipsis, max-width записей 320px).
3. **Оптимизация RAM/CPU:** коалесинг agent-событий (1 setState/33мс вместо N), окно рендера ленты (80 сообщений + «показать раньше»), кап вывода инструментов 120k символов (голова+хвост), кэш аналитики per-file, кэш подсветки ≤150, узкие zustand-селекторы.
4. **Настройки:** карточка установки pi (авто-детект / ручной путь `set_pi_path` с валидацией `--version`, поле `piPath` в конфиге), вкладки скрываются без pi; структурный редактор провайдеров models.json (+ probe эндпоинта через curl); вкладка «Расширения» с рекомендуемыми (permission-system, plannotator) и установкой в 1 клик; форма компакции (enabled/reserveTokens/keepRecentTokens).
